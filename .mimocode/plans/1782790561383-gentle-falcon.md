# Plan: Forgot/Reset Password Feature

## Context

Users currently have no way to reset a forgotten password. Better Auth has built-in support for password reset via `sendResetPassword` callback and client methods `requestPasswordReset` / `resetPassword`. Email delivery uses Brevo SMTP via nodemailer (already configured).

## Changes

### 1. Server — Add `sendResetPassword` to auth config

**File**: `src/lib/auth.ts`

Add `sendResetPassword` callback inside the `emailAndPassword` config block. Import `generateForgotPasswordEmailHtml` from `email.ts`. Use `void sendEmail(...)` (fire-and-forget pattern matching existing verification email).

### 2. Email template — Create `ForgotPasswordEmail` component

**File**: `src/components/emails/forgot-password-email.tsx` (new)

React Email component matching the verification email style (same red branding, same layout). Props: `{ url: string }`. Copy structure from `verification-email.tsx` but with reset-password copy.

### 3. Email helper — Add `generateForgotPasswordEmailHtml`

**File**: `src/lib/email.ts`

Add a new exported function `generateForgotPasswordEmailHtml(url: string)` that creates the `ForgotPasswordEmail` component.

### 4. Client — Export `forgotPassword` and `resetPassword`

**File**: `src/lib/auth-client.ts`

Better Auth client already has `forgotPassword` and `resetPassword` methods on `authClient`. Export them:
```ts
export const { ..., forgotPassword, resetPassword } = authClient;
```

### 5. Page — `/forgot-password`

**File**: `src/app/forgot-password/page.tsx` (new)

Simple page with email input form. Calls `authClient.forgotPassword({ email, redirectTo: "http://localhost:3000/reset-password" })`. Shows success message "Check your email for a reset link" after submission.

### 6. Page — `/reset-password`

**File**: `src/app/reset-password/page.tsx` (new)

Page that reads `token` from URL search params. Shows new password + confirm password form. Calls `authClient.resetPassword({ newPassword, token })`. On success redirects to `/login` with success toast. On error (invalid/expired token) shows error message.

### 7. Sign-in form — Add "Forgot password?" link

**File**: `src/components/auth/sign-in-form.tsx`

Add a "Forgot password?" link below the password field, linking to `/forgot-password`. Place it between the password field and the submit button.

### 8. Environment — Update `.env.example`

**File**: `.env.example`

Replace stale `RESEND_API_KEY` line with the actual SMTP vars used by the code: `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`.
