# Email Verification on Registration

## Overview
Add email verification to the registration flow using Resend for email delivery and Better Auth's built-in email verification support.

## Files to Create/Modify

### 1. Install Resend
- Run `npm install resend`

### 2. Create email utility: `src/lib/email.ts`
- Initialize Resend client with `process.env.RESEND_API_KEY`
- Export a `sendVerificationEmail` helper that sends the verification link
- Use `void` (no await) to prevent timing attacks per Better Auth docs

### 3. Modify `src/lib/auth.ts`
- Import `sendVerificationEmail` from `./email`
- Add `emailVerification` config block with `sendVerificationEmail` callback
- Add `requireEmailVerification: true` to `emailAndPassword` config
- Set `sendOnSignUp: true` and `autoSignInAfterVerification: true`

### 4. Create verification page: `src/app/verify/page.tsx`
- Client component that reads `token` from URL search params
- Calls `authClient.verifyEmail({ token, callbackURL })` on mount
- Shows loading state, success redirect to `/dashboard`, or error message
- Handle `invalid_token` and expired token cases

### 5. Modify `src/components/auth/register-form.tsx`
- After successful signup, show a "Check your email" message instead of redirecting to login
- Store the email in state to display which address the verification was sent to
- Add "Resend verification email" button that calls `authClient.sendVerificationEmail`

### 6. Modify `src/components/auth/sign-in-form.tsx`
- Handle 403 status error (email not verified) with a clear message
- Add "Resend verification email" link when the 403 error occurs

### 7. Modify `src/lib/auth-client.ts`
- Export `sendVerificationEmail` and `verifyEmail` from the auth client

### 8. Update `.env.example`
- Add `RESEND_API_KEY=""` and `EMAIL_FROM="DevVault <noreply@yourdomain.com>"`

## Verification
1. Run `npm run build` to check for type errors
2. Test registration flow: signup → check email → click link → account verified
3. Test sign-in without verification → see 403 error with resend option
4. Test resend verification email functionality
