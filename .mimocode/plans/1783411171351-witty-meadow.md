# Fix CORS Error: Production Frontend Calling localhost

## Root Cause

In `src/lib/auth-client.ts:4`:
```ts
baseURL: process.env.NEXT_PUBLIC_BETTER_AUTH_URL || "http://localhost:3000",
```

The env var `NEXT_PUBLIC_BETTER_AUTH_URL` is **not set in Vercel production**, so the client-side auth SDK falls back to `http://localhost:3000`. The production frontend then tries to make API calls to your local dev server, which triggers CORS.

**Why it happens**: `.env.production` has `BETTER_AUTH_URL` (server-side, no `NEXT_PUBLIC_` prefix) but is missing `NEXT_PUBLIC_BETTER_AUTH_URL` (client-side, must be `NEXT_PUBLIC_` prefixed to be bundled into browser code).

## Fix (2 changes)

### 1. Add `NEXT_PUBLIC_BETTER_AUTH_URL` to `.env.production`

**File**: `.env.production`  
Add after the existing `BETTER_AUTH_URL` line:
```
NEXT_PUBLIC_BETTER_AUTH_URL="https://devvault-eight.vercel.app"
```

### 2. Set the env var in Vercel dashboard

In Vercel project settings → Environment Variables, add:
- **Key**: `NEXT_PUBLIC_BETTER_AUTH_URL`
- **Value**: `https://devvault-eight.vercel.app`
- **Environments**: Production, Preview

This is critical because `.env.production` files are **not uploaded to Vercel** — Vercel reads env vars from its own dashboard.

## Files Modified

- `.env.production` — add `NEXT_PUBLIC_BETTER_AUTH_URL`
