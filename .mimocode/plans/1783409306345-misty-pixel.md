# Plan: Fix Seeder to Handle Existing Demo User

## Goal
Make `prisma/seed.ts` idempotent for the demo user — if the user already exists but is missing the credential account, clean up the user's children data and re-seed. If the account already exists, do nothing.

## Behavior Matrix

| State | Action |
|-------|--------|
| User missing | Create user + account + all data (current behavior) |
| User exists, account exists | Do nothing — skip all seeding |
| User exists, account missing | Delete items, collections, account → re-seed account + children |

## Changes to `prisma/seed.ts`

### 1. Add a cleanup step after the user upsert

After line 30, before seeding item types, add logic to check for the credential account:

```typescript
// Check if the credential account exists
const existingAccount = await prisma.account.findFirst({
  where: {
    userId: user.id,
    accountId: "demo@devvault.io",
    providerId: "credential",
  },
});

if (existingAccount) {
  console.log("Demo user and account already exist — skipping seed");
  return; // or skip to end
}

// Account missing — clean up children and re-seed
await prisma.itemCollection.deleteMany({
  where: { item: { userId: user.id } },
});
await prisma.item.deleteMany({ where: { userId: user.id } });
await prisma.collection.deleteMany({ where: { userId: user.id } });
await prisma.account.deleteMany({ where: { userId: user.id } });
console.log("Cleaned up existing demo user data");
```

### 2. Change user upsert to only upsert the user record

The current `upsert` includes `accounts.create` in the `create` clause — this is skipped when the user already exists. Since we now handle account creation separately (after cleanup), we should:

- Use `upsert` with only user fields in both `create` and `update` (or just ensure the user exists with `findFirst` + `create`)
- Create the account separately after cleanup, so it always runs when needed

Simplified user handling:
```typescript
let user = await prisma.user.findUnique({
  where: { email: "demo@devvault.io" },
});

if (!user) {
  user = await prisma.user.create({
    data: {
      email: "demo@devvault.io",
      name: "Demo User",
      emailVerified: true,
      isPro: false,
    },
  });
}
```

### 3. Move account creation to after cleanup

Create the credential account as a standalone step, after cleanup but before items/collections:

```typescript
const hashedPassword = await hashPassword("12345678");
await prisma.account.create({
  data: {
    accountId: "demo@devvault.io",
    providerId: "credential",
    userId: user.id,
    password: hashedPassword,
  },
});
console.log("Seeded credential account");
```

### 4. Keep system item types as-is

System item types (Snippet, Prompt, etc.) are user-agnostic (`userId` is null). They use their own upsert with `update: { icon }` which correctly updates icons if they changed. No changes needed here.
