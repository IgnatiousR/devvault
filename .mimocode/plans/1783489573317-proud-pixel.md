# Fix: Create dialog type not preselected correctly on first open

## Root Cause

In `src/components/dashboard/create-item-dialog.tsx`, line 57:

```tsx
const [selectedType, setSelectedType] = useState("snippet")
```

`selectedType` is hardcoded to `"snippet"` on mount, ignoring the current URL. There's a `getDefaultType(pathname)` function that correctly derives the type from the URL, but it's only called in `resetForm()` — which runs when the dialog **closes**, not when it first opens.

So on first open from `/items/command`, the type shows "snippet" (wrong). On close, `resetForm()` sets it to "command". On second open, the state already has "command".

## Fix

Use a lazy initializer for `useState` so `getDefaultType(pathname)` runs on the first render:

```tsx
const [selectedType, setSelectedType] = useState(() => getDefaultType(pathname))
```

`usePathname()` is called before this `useState` (line 56), so `pathname` is available at initialization time.

## File to modify

- `src/components/dashboard/create-item-dialog.tsx` — change line 57

1. Navigate to `/items/command`
2. Click "New Item" — type should be preselected as "command"
3. Close the dialog
4. Navigate to `/items/snippet`
5. Click "New Item" — type should be preselected as "snippet"
