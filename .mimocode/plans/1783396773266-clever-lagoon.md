# GitHub OAuth — Force Account Picker

## Problem

GitHub OAuth auto-redirects if the user is already logged in to GitHub and has previously authorized the app. Users can't choose a different account.

## Fix

Add `prompt: "login"` to the GitHub provider config. This forces GitHub to show the login page every time.

### File to modify

`src/lib/auth.ts` — add `prompt: "login"` to the `github` socialProvider config:

```ts
socialProviders: {
  github: {
    clientId: process.env.GITHUB_CLIENT_ID!,
    clientSecret: process.env.GITHUB_CLIENT_SECRET!,
    prompt: "login",  // force account picker
  },
},
```

### Verification

- Visit `/login`, click "Continue with GitHub"
- GitHub should show the login/account-picker page instead of auto-redirecting
- If user has multiple GitHub accounts, they can choose which one to use
